HOW TO RELEASE UPDATES — Slippi Ranked Stats
============================================

ONE-TIME SETUP (already done, but documented here for reference)
-----------------------------------------------------------------
1. The signing keypair has been generated (srs.key + srs.key.pub).
   srs.key is in .gitignore and must NEVER be committed.

2. Add the private key to GitHub:
   - Open srs.key in a text editor and copy all contents
   - Go to: https://github.com/Joey-Farah/Slippi-Ranked-Stats/settings/secrets/actions
   - Click "New repository secret"
   - Name:  TAURI_SIGNING_PRIVATE_KEY
   - Value: paste the contents of srs.key
   - Click "Add secret"

3. The GitHub Actions workflow is already set up at:
   .github/workflows/release.yml
   It runs automatically whenever you push a version tag.


RELEASING A NEW VERSION (do this every time)
--------------------------------------------
1. Make your code changes.

2. Bump the version number in ALL THREE of these files:
   - src-tauri/tauri.conf.json   → "version": "x.x.x"
   - src-tauri/Cargo.toml        → version = "x.x.x"
   - package.json                → "version": "x.x.x"

3. Copy the updated project files into your GitHub-connected folder
   (make sure to include .github/workflows/release.yml).

4. Commit and push to GitHub:
   git add .
   git commit -m "Release v1.1.8"
   git push

5. Create and push a version tag:
   git tag v1.1.8
   git push origin v1.1.8

6. GitHub Actions will automatically:
   - Build the app
   - Sign the installer
   - Create a GitHub Release with the .exe installer
   - Publish latest.json so existing installs auto-update

7. Users with the app already installed will see an "Update available"
   banner the next time they open the app and can install with one click.


VERSION NAMING CONVENTION
--------------------------
Use semantic versioning: MAJOR.MINOR.PATCH
  - PATCH (x.x.1): Bug fixes, small tweaks
  - MINOR (x.1.x): New features
  - MAJOR (1.x.x): Large overhauls or breaking changes

Current version: 1.1.8


NOTES
-----
- The private key (srs.key) is stored locally at:
  slippi-dashboard-v2/srs.key
  Keep this file safe. If you lose it you cannot sign future updates
  and users will need to manually reinstall.

- The public key is embedded in src-tauri/tauri.conf.json under
  plugins.updater.pubkey — do not change this.
